Web Servers

A designated home page is often the index file. This is the default file that is served when a user requests the root directory of a website, and it usually contains the main content of the website. The name of the designated home page varies depending on the server and language used. For example, index.html or index.php is often the designate homepage in Apache servers.

The browser's Web Dev/Inspector Network tab displays the files that are being requested and loaded by the browser. These files are known as resources. Network tab displays the age of the file, its cache details, server date, last-modified date, server type, and code read by the server.

The file details are different when viewed from the web server because the file details may include additional information such as the status of the file, HTTP headers, server configurations, and permissions.

The favicon.ico file has a status 200 because the file was successfully found and loaded by the browser. The main.css and main.js files have a status 400 because the files are not found and cannot be loaded by the browser.

The scheme is the protocol that is used to communicate with the client. The subdomains are the subdomains of the host domain. The host domain is the domain of the web server. The resources are the resources that are being loaded by the browser. The parts of the web server's URL are as follows, the scheme: https://, the subdomains: web.engr., the domain: oregonstate.edu, and the resources: /~higgirac/a1-higgirac/ .

Frontend Design

Frontend Design is the process of creating the user interface and user experience of a website or web application. This involves using technologies such as HTML, CSS, and JavaScript to build the visual and structural elements of a web page. HTML (HyperText Markup Language) is used to define the structure, including paragraphs, headings, links, and images among other content elements. CSS (Cascading Style Sheets) is applied to HTML for stylistic decision making in regards to these elements. Finally, JavaScript is used to create web pages' dynamic behavior. Frontend design also involves usability and inclusivity considerations in the design and engineering process. These concepts when applied in frontend design work together to build interfaces that are visually appealing, and user-friendly.

Effective:
A website or product’s capability of facilitating users in achieving their objectives without encountering significant errors or inaccuracies.
Efficient:
The ability of users to accomplish tasks in the shortest possible timeframe and with minimal effort.
Easy to Navigate:
Ensures that users can effortlessly find their desired content or functionalities, particularly with those unfamiliar with the interface.
Error-free:
The aim of identifying and addressing common user errors is to eliminate and minimize issues that can negatively impact the accessibility and availability to desired resources for the user.
Enoyable to Use; Engaging:
Fosters user engagement and satisfaction by satisfying the unique needs and preferences of the intended audience, and by effectively incorporating the previous 4 "E"s into the product design process.

The <header> tag is used to define the introductory section of a webpage which typically includes the name, publisher, and marketing slogan. The <main> tag is used to define the main content area. The <nav> tag is used to define a section of navigation links allowing users to navigate to different pages or sections within a website. Main menu, search, tools, stories, locations, and legal links are typically included in sets of navigation. The <section> tag is used to divide a webpage into content blocks. It organizes related, but not standalone, content into meaningful portions. The <article> tag defines standalone content which can can be distributed and reused independently. The <footer> tag defines the bottom section of a webpage which often contains copyright information, contact information, and links to critical pages.

  1. Linking to External Content allows for easy navigation to external content. This requires the Anchor element <a> to create a hyperlink that specifies the URL of the external website using the href attribute. For example, <a href=“https://example.com">External Link</a>.
  2. Linking to Internal Content is the act of linking to internal content within the same webpage by using Anchor tags with href attributes pointing to IDs which define the reference for an anchor. For example, the id: "<article id=“Introduction”>The content for this section…</article>", and the anchor: <a href=“#Introduction”>Introduction</a>.
  3. Linking Page-to-Page Content is the creation of links to other webpages within the same website, creating page-to-page navigation. This is accomplished by specifying the relative (for internal resources) or absolute (for external resources) URL of the target page in the href attribute. For example, <a href=“../images/logos.png”>Image logo</a>, or <a href=“https://example.com/app/images/logos.png">Image logo</a>.

Optimizing Images

There are six major specifications for optimizing images:

  1. Descriptive File Name: Enhances search engine optimization (SEO) by making the image content clear and searchable. Using specific descriptors helps search engines categorize and display the image appropriately in search results.
  2. Small File Size: Improves website loading times by reducing the amount of data transferred when a page is accessed. This is achieved through compression techniques, which can either be lossy (reducing quality) or lossless (maintaining quality).
  3. Exact Dimensions: Ensures that images fit precisely within the designated space on a webpage without needing to be resized by the browser, which can slow down page load times and degrade image quality.
  4. Correct File Format: Selecting the right file format (e.g., JPG, PNG, GIF) based on the nature of the image and its use on the web optimizes the balance between quality and file size. Different formats are suited for different types of images and uses.
  5. Reduced Resolution: Matches the image resolution to the capabilities of most viewing devices, which is typically lower than the resolution used for printing. This avoids unnecessary high resolution that can increase file size without a visible improvement in quality on standard displays.
  6. Color Mode: Chooses the appropriate color mode (RGB or Indexed) for the image based on how it will be displayed. RGB is used for most images due to its broad color range, while Indexed is useful for simpler images to reduce file size.

Different image file formats are optimized for specific types of content due to their unique compression methods and color support. Photos, which usually contain complex color gradients and details, are best served by the JPEG (JPG) format. JPEG offers a good balance between quality and file size through adjustable compression that reduces file size while maintaining accuracy. For line art, such as logos or simple graphics with fewer colors, the GIF format is suitable because it supports a limited color palette (up to 256 colors), which helps keep the file size small. Additionally, PNG can be a good choice for line art that requires high clarity with support for transparency and lossless compression, preserving the sharpness of edges without degradation over multiple edits.

Browsers and devices utilize Favicons to improve user navigation and strengthen brand identity across various interfaces. These small icons appear in browser tabs, assisting users in quickly identifying and switching between sites. Favicons are also used in bookmarks and browser history, offering a visual sign that makes specific pages easier to find. Furthermore, when websites are saved to a smartphone’s home screen or as app shortcuts, favicons serve as the application icons, enhancing recognition. Some search engines display favicons in search results, further aiding brand visibility and user orientation.

Cacading Stylesheets

Stylesheets, typically CSS (Cascading Style Sheets), are essential for websites and apps because they enable developers to separate content from design, allowing for more efficient and flexible styling across multiple web pages. This separation simplifies maintaining and updating the visual appearance of a site or app. Stylesheets enhance user experience by providing consistent theming and responsive design, ensuring that websites and apps look good on various devices and screen sizes. Additionally, CSS can improve load times and performance by reducing the need for inline styling and repetitive code.

Incorporating styles in web development can be achieved through five main methods, each offering unique advantages. External stylesheets are used widely for their efficiency in applying consistent styles across multiple pages via a single CSS file, making maintenance and updates easier. Imported CSS in a global stylesheet allows modular management of styles but can increase load times due to additional HTTP requests. Embedded CSS within an HTML file using the <style> tag is useful for page-specific styles, simplifying deployment without affecting other pages. Inline styles applied directly to HTML elements with the style attribute are good for quick, one-off modifications, although they hinder style reusability and clutter HTML. Lastly, JavaScript-based styling manipulates styles dynamically which is ideal for interactive and condition-based styling, enhancing user experiences by adapting to runtime conditions.

Forms

The six primary goals of accessible forms:

  1. Provide clear instructions: Clear instructions above the form and in the labels helps users understand how to interact with the form and provide the required information accurately.
  2. Inform users about data collection and required fields: Clear communication about why certain data is being collected and which fields are mandatory helps users understand the form's purpose and requirements.
  3. Set the first field to autofocus: Autofocusing the first field allows users to start typing immediately without manually selecting the input field, enhancing usability and efficiency.
  4. Accessible via keyboard: Keyboard accessibility ensures that users who cannot or prefer not to use a mouse can navigate and interact with every form effectively.
  5. Tab indexing to complex forms: Tab indexing specifies the order in which form fields are navigated when using the tab key, improving the logical flow and usability of complex forms.
  6. Ensure validation messages are screen-readable: Screen-readable validation messages ensure that users, including those using screen readers, receive clear and understandable feedback about errors or validation issues with their form submissions.

Major HTML tags used in forms:

  • <form>: Defines a document section containing form elements. Attributes include "action" (specifies where the form data is sent) and "method" (specifies HTTP method for submitting form data).
  • <fieldset>: Groups related form controls together, which helps with accessibility and organization. It can contain one or more <legend> tags for labeling the group.
  • <legend>: Provides a caption or label for a <fieldset> element, helping users understand the purpose of the grouped form controls.
  • <label>: Associates a label with a form control, improving accessibility and usability. Uses the "for" attribute to link to the "id" of the associated control.
  • <input>: Represents an input control, such as text input, checkbox, radio button, etc. Attributes include "type" (specifies the type of input), "name" (defines the name of the input field), and "required" (indicates if the field is mandatory).
  • <select>: Creates a dropdown list of options. Contains one or more <option> tags for specifying the available choices. Attributes include name (defines the name of the select field) and "multiple" (allows multiple selections if present).
  • <textarea>: Defines a multiline text input control. Attributes include "name" (defines the name of the textarea field) and "rows"/"cols" (specifies the visible number of rows/columns).
  • <button>: Defines a clickable button often used for form submission. It can have attributes like "type" (specifies the type of button, e.g., "submit" or "reset") and "disabled" (turns off the button).

Node / Express

Node.js, npm, and Express.js are essential technologies in modern web development.

Node.js is a runtime environment that allows developers to run JavaScript code outside a web browser and use JavaScript on the client and server sides, allowing full-stack JavaScript development. Due to its non-blocking and event-driven architecture, Node.js is highly scalable and capable of efficiently handling many concurrent connections. Node.js offers developers the ability to build scalable and high-performance applications efficiently.

npm (Node Package Manager) is the default package manager for Node.js. It installs, manages, and shares packages of JavaScript code. npm provides access to an extensive ecosystem of open-source libraries and tools that can be easily integrated into Node.js projects. npm simplifies dependency management and allows developers to leverage existing solutions to accelerate development.

Express.js is a web application framework for Node.js designed to build web applications and APIs. It provides robust features for handling HTTP requests and responses, routing, middleware integration, templating engines, and more. Express.js follows the middleware pattern, allowing developers to combine modular components to create powerful and flexible applications.

These three technologies improve modern web development, allowing developers to build efficient, scalable, and flexible web applications and APIs.

JavaScript

JavaScript has several fundamental data types that are used to manipulate variables and store values. These include primitive data types like strings, numbers, symbols, and booleans, representing textual data, numerical data, unique identifiers, and true/false values. A Symbol is a unique and immutable data type introduced in ES6 to create unique identifiers for object properties. Its uniqueness makes symbols ideal for defining object properties that must be private or protected from accidental changes or collisions in large codebases. There are also null and undefined, which signify the absence of a meaningful value and an uninitialized state. JavaScript also supports complex data types like objects, which are collections of properties mapped by strings or symbols and can contain various data types. Objects serve as a fundamental building block in JavaScript and can be used to create more complex structures like arrays, functions, and more. A Function is another complex data type. Functions are blocks of code designed to perform a particular task or calculate values and can be invoked throughout the program.

Objects in JavaScript are used to store keyed collections of various data and more complex entities. An object is created by listing properties inside curly braces {}, with each property being a key-value pair. Arrays are unique kinds of objects used for storing ordered collections of items, accessible by indices. JSON (JavaScript Object Notation) is a lightweight data-interchange format that uses human-readable text to store and transmit data objects consisting of attribute-value pairs. It is heavily used for data exchange between a server and web applications.

Conditionals and Loops are used in JavaScript to control the flow of execution based on different conditions and to perform repetitive tasks, respectively. Conditionals like if, else, and switch allow the code to execute different blocks based on various conditions. Loops, such as for, while, and do-while, enable executing a code block multiple times until a specified condition fails. These structures are fundamental for creating dynamic behavior in applications.

Object-oriented programming (OOP) is a programming paradigm based on the concept of "objects", which can contain data, in the form of fields (often known as attributes or properties), and code, in the form of functions (often known as methods). In JavaScript, OOP is implemented using prototypes and classes. Classes provide a more straightforward and cleaner way to create objects and deal with inheritance, allowing more complex and modular software development.

Functional programming is a programming paradigm where programs are constructed by applying and composing functions. Functional programming is declarative and avoids shared state, mutable data, and side effects. JavaScript supports functional programming by treating functions as having special privileges, meaning that functions can be assigned to variables, passed as arguments, and returned from other functions, enabling a flexible approach to software construction.
